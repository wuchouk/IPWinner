# Streamlit Cloud 免費部署說明

只要 3 個步驟，就能讓同事透過網址使用這個工具。

---

## 步驟 1：上傳到 GitHub

1. 登入 https://github.com （沒有帳號的話免費註冊一個）
2. 點右上角 **「+」→「New repository」**
3. 取個名稱，例如 `trademark-merge-tool`，設定為 **Private**（只有你能看到原始碼）
4. 建立後，將 `streamlit-app` 資料夾裡的 3 個檔案上傳到 repo：
   - `streamlit_app.py`
   - `requirements.txt`
   - `.streamlit/config.toml`

   > 最簡單的方式：在 GitHub repo 頁面點 **「Add file → Upload files」**，把檔案拖進去

---

## 步驟 2：部署到 Streamlit Cloud

1. 前往 https://share.streamlit.io
2. 用你的 GitHub 帳號登入
3. 點 **「New app」**
4. 填寫：
   - **Repository**：選擇剛才建立的 `trademark-merge-tool`
   - **Branch**：`main`
   - **Main file path**：`streamlit_app.py`
5. 點 **「Deploy!」**

等待約 1-2 分鐘，部署完成後你會得到一個網址，例如：
```
https://trademark-merge-tool.streamlit.app
```

---

## 步驟 3：分享給同事

把網址傳給同事，他們打開瀏覽器就能直接使用，不需要安裝任何軟體。

---

## 常見問題

**Q: 要花錢嗎？**
A: 不用。Streamlit Community Cloud 是免費的。

**Q: 有流量限制嗎？**
A: 免費版適合團隊內部使用，沒有嚴格的流量限制。如果長時間沒人使用，app 會自動休眠，下次打開時會花幾秒鐘啟動。

**Q: 上傳的檔案安全嗎？**
A: 上傳的檔案只在處理期間暫存於伺服器記憶體中，不會被永久保存。合併完成後下載即可。

**Q: 可以限制只有特定人能用嗎？**
A: 可以。在 Streamlit Cloud 的 app 設定中，可以設定 **Viewer authentication**，指定允許的 email 清單。

**Q: 我想改欄位對應怎麼辦？**
A: 修改 `streamlit_app.py` 裡最上方的 `DB1_MAPPING`、`DB2_MAPPING`、`DB3_MAPPING` 字典，Push 到 GitHub 後 Streamlit Cloud 會自動重新部署。
