# IP Winner Email Processor V3 — 分類測試與成本效益分析報告

> 測試日期：2026-03-13
> 測試資料：1,738 封 eml（整理好的 1,250 封 + 下載區 488 封）

---

## 一、測試方法

以 V3 設計規則為基礎，用 Python 腳本解析所有 eml 檔案的**實際內容**（非檔名），執行：

1. **規則引擎前處理**：抽取案號、判斷 F/T 方向、查 sender role
2. **收發碼判定**：根據 sender/recipient domain 對照名單
3. **準確率比對**：與客戶手動整理的 1,249 封 ground truth 比較

此測試**僅用規則引擎**（無 LLM），目的是建立 baseline 並估算 token 用量。

---

## 二、收發碼準確率

### 初始名單（33 個 domain）

| 指標 | 數值 |
|------|------|
| 測試信件數 | 1,249 封（有 ground truth） |
| 正確數 | 1,196 封 |
| 準確率 | **95.8%** |
| 錯誤數 | 53 封 |

錯誤主因：**39 封（74%）是 sender/recipient domain 不在名單**，屬於名單不完整的問題。

### 補完名單後（41 個 domain）

補入 8 個遺漏 domain 後：

| 指標 | 數值 |
|------|------|
| 正確數 | 1,234 封 |
| 準確率 | **98.8%** |
| 剩餘錯誤 | 15 封（1.2%） |

需補入的 domain：

| Domain | 角色 | 說明 |
|--------|------|------|
| naipo.com | A（代理人） | |
| smartbiggar.ca | A（代理人） | Smart & Biggar - 加拿大 |
| bereskinparr.com | A（代理人） | Bereskin & Parr - 加拿大 |
| cpaglobal.com | A（代理人） | CPA Global |
| altitude-ip.com | A（代理人） | Altitude IP - 加拿大 |
| kankrishme.com | A（代理人） | Kankrishme - 印度 |
| perkinscoie.com | C（客戶） | Perkins Coie |

### 剩餘 15 封錯誤分析

這些是**需要 LLM 語義理解**才能正確分類的邊界案例：

| 錯誤類型 | 數量 | 說明 |
|---------|------|------|
| TA→FA 或 FA→TA | 8 封 | sender 是代理人但 email 方向需看內容才能判定 |
| FC→TC | 2 封 | 客戶來信但被當成寄出 |
| FA→TC | 2 封 | 代理人來信被當成寄給客戶 |
| FA→FG | 1 封 | 代理人轉寄政府文件，被誤判為政府來信 |
| TG→FG | 1 封 | 寄給政府被當成政府來信 |
| TA→TX | 1 封 | 寄給代理人但 recipient 不在名單 |

**結論：純規則引擎就能達到 98.8%，LLM 只需處理剩餘 1.2% 的邊界案例。**

## 六、AI vs 人力成本效益分析

### 人工處理時間估算

每封 email 的人工處理步驟：

| 步驟 | 時間（分鐘） |
|------|-------------|
| 開啟並閱讀 email 內容 | 1.5 |
| 判斷收發碼（查看 sender、內容） | 0.5 |
| 提取案號 | 0.5 |
| 決定語義檔名 | 1.0 |
| 下載 eml + 附件並歸檔 | 1.5 |
| 多案號時複製到各資料夾 | 0.5 |
| **每封平均** | **5.5 分鐘** |

### 月工作量

| 指標 | 數值 |
|------|------|
| 每日信件量 | ~100 封 |

## 七、結論與建議

### 測試結論

1. **規則引擎 baseline 準確率 98.8%** — sender/recipient domain 比對極有效
2. **LLM 只需處理 1.2% 邊界案例** — 成本極低
3. **月 AI 成本約 NT$160**（Gemini 3.0 Flash）vs 人力 NT$40,000+
4. **Token 用量比預期低** — 平均每封 ~2,830 tokens，原估偏高

### 建議行動

1. **立即補完 sender 名單**：加入 7 個遺漏 domain
2. **修正設計文件成本估算**：從 $7.5/月 → $4.92/月
3. **開始 Phase 1 實作**：先用規則引擎 + Gemini 3.0 Flash
4. **建立 validation 機制**：先跑 1 週人工抽查，確認品質後再全自動化
